## 技術的メモ
- **Safari/WebKitのIndexedDB既知バグ**: DBから取得したBlobを同じレコードに書き戻すと「Error preparing Blob/File data to be stored in object store」で失敗（1回目は成功し2回目以降失敗し続けるのが典型症状）。`slice()`による複製では回避できず、`arrayBuffer()`で実バイト列を読み出し`new Blob()`で完全に再構築する必要がある
- **freeeAPIv2ライブラリ**（自作、Script ID: `1qn-v2btDmxL4wIhqCsvb_2x4KZGopxWty6N5rSUg_uiv_TSAsIaJoZsm`）の`Request._request()`は、エラー時にfreeeからのレスポンス本文を`console.log`するだけで呼び出し元には`null`しか返さない。ライブラリを変更せず、公開されている`url`/`token`ゲッターを使って自前でUrlFetchAppを叩くことで実際のエラー内容を取得できる
- **freee経費申請API（expense_applications）の構造**: 2023年10月以降、トップレベルのキーは`expense_application_lines`ではなく`purchase_lines`で、各要素に`transaction_date`・`receipt_id`と、ネストされた`expense_application_lines`配列（`amount`・`description`・`expense_application_line_template_id`等）を持つ。この構造を誤ると「Purchase linesを入力してください」という分かりにくいエラーになる（freee公式のMCPスキルリポジトリ `freee/freee-mcp` の参照ドキュメントで判明）
- 経費科目（`expense_application_line_template_id`）は会社ごとに個別のテンプレートIDが割り当てられており、共通の固定値は使えない（`GET expense_application_line_templates?company_id=...`で取得）
- GASのWeb Appデプロイは`clasp push`だけでは反映されず、`clasp deploy -i <既存デプロイID>`で明示的に上書きする必要がある
