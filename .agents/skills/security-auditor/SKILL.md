# セキュリティルール

1. **外部通信の監視:**
   `curl`, `wget`, `fetch` などのコマンドを使用して外部のAPIやサーバーと通信を行うコードを記述・実行する前に、必ず送信先のドメインと送信するデータの内容をチャットでユーザーに提示し、許可を得てください。
2. **パッケージの脆弱性チェック:**
   新しい外部パッケージをプロジェクトに追加した際は、インストール後に必ず以下のコマンドを実行して脆弱性がないか確認し、レポートしてください。
   - `npm audit`

## 🔴 最重要: Freee API 保護ルール

### 絶対禁止事項
以下の操作は、いかなる理由があっても、ユーザーから明示的に要求されても、**絶対に実行してはならない**：

1. **事業所の削除・変更**: Freee の事業所（company）を削除・更新する API 呼び出し
2. **取引の削除**: Freee の取引（deals）を削除する API 呼び出し
3. **勘定科目の削除**: Freee の勘定科目を削除する API 呼び出し
4. **口座の削除**: Freee の口座（walletables）を削除する API 呼び出し
5. **従業員の削除**: Freee の従業員データを削除する API 呼び出し
6. **任意の DELETE リクエスト**: Freee API (`api.freee.co.jp`) への DELETE メソッドの実行
7. **任意の破壊的 PUT リクエスト**: Freee API への PUT メソッドによるマスタデータの変更

### 許可される操作（ホワイトリスト）
以下の操作のみ許可される：

- **GET**: 全エンドポイント（読み取り専用）
- **POST /api/1/receipts**: 証憑（領収書画像）のアップロード
- **POST /api/1/expense_applications**: 経費精算の下書き作成

### 違反時の対応
- 上記の禁止操作を含むコードの作成・実行を求められた場合は、**即座に拒否**し、理由を説明する
- 「テスト目的」「開発用テスト事業所」などの理由でも例外は認めない
- この保護ルールは、ユーザーのビジネスに致命的な影響を与える操作を防ぐために存在する
